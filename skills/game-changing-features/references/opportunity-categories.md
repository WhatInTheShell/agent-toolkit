# Opportunity Category Sweep

Force yourself through each category before declaring the list exhaustive:

| Category | Question | Practitioner trap |
|----------|----------|-------------------|
| **Speed** | What takes too long? | Don't stop at "make it faster" — find the step users skip entirely because it's too slow |
| **Automation** | What's repetitive? | Users often don't know they're doing repetitive work; look for copy-paste patterns in sessions |
| **Intelligence** | What could be smarter? | Recommendations only work if users trust them; check if trust is the real problem first |
| **Integration** | What else do users use? | Integrations that require setup > 5 min see <20% adoption; count setup steps before committing |
| **Collaboration** | How do users work together? | Collaboration features used by 1 person on a team create resentment, not value |
| **Personalization** | How is everyone different? | Personalization that requires user configuration > 3 steps almost never gets used |
| **Visibility** | What's hidden that shouldn't be? | Surface insights only if users can act on them; dashboards no one acts on erode trust |
| **Confidence** | What creates anxiety? | Undo, confirmations, and previews lower support tickets more than any "delight" feature |
| **Delight** | What could spark joy? | Delight that fires on tasks < 30s/day has near-zero retention impact — pick high-frequency moments |
| **Access** | Who can't use this yet? | Mobile-first expansion that degrades desktop experience costs you both segments |

# Output Template

```markdown
# 10x Analysis: <Product/Area>
Session N | Date: YYYY-MM-DD

## Current Value
What the product does today and for whom. One paragraph max.

## Highest-Leverage Moves

### Do Now (Quick wins — ship within 2 weeks)
1. [Feature] — Why: [reason] | Impact: [what changes] | Evidence: [what you observed]

### Do Next (High leverage — 1-2 months)
1. [Feature] — Why: [reason] | Unlocks: [what becomes possible]

### Strategic Bets (Transformative — 3+ months)
1. [Feature] — Upside: [what we gain] | Risk: [what could go wrong] | Signal to watch: [leading indicator]

### Backlog (Good but not now)
1. [Feature] — Why later: [reason]

## Open Questions
- [ ] Validate assumption: ...
- [ ] Research: ...
- [ ] Decide: ...
```

# Unstick Prompts

If ideation stalls after 10 minutes, cycle through these:
- "What would make a user tell their friend about this tonight?"
- "What do power users do manually that we could make native?"
- "What would we build if we had 10x the team? 1/10th the team?"
- "What would a competitor need to ship to make us irrelevant in 18 months?"
- "What insight do we have from usage data that users themselves don't see?"
- "What's the feature that sounds crazy but has a non-zero chance of being transformative?"
- "What anxiety do users have at 11pm before a deadline that we could eliminate?"
